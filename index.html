How to Use This Checklist

This checklist covers the data elements required for a valid Louisiana Incident-Based Reporting System (LIBRS) submission to the LCLE and FBI. Work through each section as you write your report. Items marked ⚠ COMMON ERROR are frequently flagged in BRPD submissions — pay extra attention to these. Use the NIBRS Code Verification Tool to confirm the correct offense code for each LRS statute charged.

All Incidents Universal Incident Requirements ▶
Required for every incident submitted — regardless of offense type or whether an arrest was made.
Incident Identification
ORI Number is correct (LA0170200)
DE 1 BRPD ORI — must match exactly on every submission segment.
Incident Number is present and unique
DE 2 Local RMS incident number — must be unique within the submission period.
Incident Date is within the reporting month
DE 3 ⚠ COMMON ERROR. Incidents from prior periods appearing in the current submission cause mismatch warnings. Verify date matches the reporting period.
Incident Time is recorded (24-hour format)
DE 3 Use 0000–2359. If unknown, use best estimate and note in narrative.
Exceptional Clearance coded (if applicable)
DE 4 Required if incident is cleared exceptionally. Valid codes: A (Death of Offender), B (Prosecution Declined), C (Extradition Denied), D (Victim Refused to Cooperate), E (Juvenile/No Custody).
Cargo Theft indicator completed (if applicable)
DE 2A Required when cargo theft occurred. Answer Y or N.
Location
Location Type selected (specific type, not generic)
DE 9 Choose the most specific applicable type. Examples: Residence/Home, Highway/Road/Alley, Convenience Store, Parking Lot/Garage. Do not default to Other/Unknown unless truly unknown.
Number of Premises Entered (if residence/hotel)
DE 10 Required only when Location Type is Hotel/Motel/Etc. or Residence and a forced entry occurred.
All Incidents Offense Segment ▶
⚠ Hierarchy Rule: When multiple offenses occur in one incident, list all offenses — do not drop lower-ranked offenses. LIBRS requires reporting all Group A offenses. Group B offenses are only reported when an arrest is made.
Offense Classification
NIBRS Offense Code verified against LRS statute
⚠ COMMON ERROR. Use the NIBRS Code Verification Tool to confirm the correct code for the charged LRS statute. Sexual offenses, homicide, and robbery are frequently miscoded.
Attempted / Completed indicator selected
DE 7 A = Attempted, C = Completed, U = Unfounded. Homicide is always Completed.
Offender Suspected of Using (alcohol/drugs/computer)
DE 8 Select all that apply: A Alcohol, C Computer/Equipment, D Drugs/Narcotics, R Drone/UAS, N Not Applicable.
Bias Motivation / Hate Crime coded
DE 8A Required for all Group A offenses. Use 88 (None/No Bias) if no bias motive. Do not leave blank.
Entry & Weapons
Method of Entry coded (burglary only)
DE 11 ⚠ COMMON ERROR. Mandatory for NIBRS 220 (Burglary/B&E). Must be F (Force) or N (No Force). Frequently left blank, causing segment rejection.
Type of Criminal Activity / Gang Info coded
DE 12 Required for drug offenses (35A, 35B), gambling, and pornography. Select all applicable types.
Type of Weapon / Force Used coded
DE 13 Required for crimes against persons (assault, robbery, homicide, kidnapping). Select all weapons used. Use 99 (Unknown) if weapon type cannot be determined. Note: 90 = Other includes deadly/dangerous/communicable diseases.
Property Property Segment ▶
Required whenever property was stolen, damaged, recovered, seized, or used in the offense. Not required for crimes against persons only (assault, etc.) unless property was also involved.
Property Loss / Description
Type of Property Loss coded
DE 14 Options: 1 None, 2 Burned, 3 Counterfeited, 4 Destroyed/Damaged, 5 Recovered, 6 Seized, 7 Stolen, 8 Unknown.
Property Description type selected
DE 15 Choose the specific property type (e.g., Automobile, Firearms, Jewelry, Purse/Handbag). Do not default to Other if a specific type applies.
Property Value is not $0 or blank for stolen/damaged property
DE 16 ⚠ COMMON ERROR. Every stolen or damaged property entry must have a dollar value greater than $0. Use best estimate from victim. $1 is acceptable minimum when value is truly unknown.
Recovery Date entered (if property was recovered)
DE 17 Required when Type of Property Loss = 5 (Recovered). Must be on or after the incident date.
Number of Stolen Motor Vehicles entered
DE 18 Required when property description is motor vehicle and loss type is Stolen.
Number of Recovered Motor Vehicles entered
DE 19 Required when property description is motor vehicle and loss type is Recovered.
Drug / Narcotic Offenses Only
Suspected Drug Type coded
DE 20 Required for 35A and 35B offenses. Select all drug types involved. Up to 3 types can be entered.
Estimated Drug Quantity and Measurement entered
DE 21 / DE 22 Enter the estimated quantity and unit of measurement for each drug type. Use XX for unknown quantity.
Persons Victim Segment ▶
Required for every offense. Businesses, government agencies, and society can also be victim types — not just individuals.
Victim Identification
Victim Type selected
DE 25 I Individual, B Business, F Financial Institution, G Government, L Law Enforcement Officer, O Other, R Religious Organization, S Society/Public, U Unknown.
Offense(s) connected to this victim are listed
DE 24 Each victim must be linked to at least one offense in the incident. Multiple victims can share the same offense.
Individual Victims Only
Victim Age is recorded (or approximate range)
DE 26 Use actual age in years. For unknown age, enter 00. For unknown age with known range, use age category codes.
Victim Sex is recorded
DE 27 M Male, F Female, U Unknown.
Victim Race is recorded
DE 28 W White, B Black/African American, I American Indian/Alaska Native, A Asian, P Native Hawaiian/Pacific Islander, U Unknown.
Victim Ethnicity is recorded
DE 29 H Hispanic/Latino, N Not Hispanic/Latino, U Unknown.
Resident Status recorded
DE 30 R Resident, N Nonresident, U Unknown.
Crimes Against Persons Only
Aggravated Assault / Homicide Circumstances coded
DE 31 ⚠ COMMON ERROR. Mandatory for 09A, 09B, 09C, and 13A. Describes the circumstances (e.g., Argument, Drug Dealing, Domestic Violence). Frequently left blank causing segment rejection.
Additional Justifiable Homicide Circumstances (if 09C)
DE 32 Required only for 09C Justifiable Homicide. Describes the circumstances of the killing.
Type of Injury coded
DE 33 Required for assault, robbery, kidnapping. Up to 5 per victim. Select all that apply: N None, B Apparent Broken Bones, G Gunshot Wound (new 2025), I Possible Internal Injury, L Severe Laceration, M Apparent Minor Injury, O Other Major Injury, T Loss of Teeth, U Unconsciousness.
Offender Number(s) connected to victim
DE 34 Link each victim to the offender(s) responsible. If offender is unknown, use 00.
Relationship of Victim to Offender coded
DE 35 Required for crimes against persons. Must have a relationship code for each victim-offender pairing (e.g., SE Spouse, AQ Acquaintance, ST Stranger, RU Relationship Unknown). New 2025 codes: CO Cohabitant (non-intimate), FP Foster Parent, FC Foster Child. ⚠ COMMON ERROR in domestic/sexual offense cases.
Law Enforcement Officer Data completed (if victim is LEO)
DE 25L If Victim Type = L, additional fields are required: Activity, Assignment, ORI of agency.
All Incidents Offender Segment ▶
At minimum, one offender must be listed even if unknown. Use a single 00 unknown offender entry when nothing is known about the offender.
At least one offender segment exists for the incident
If offender is completely unknown, enter a single offender with Sequence Number 00 and all demographic fields as Unknown.
Offender Age recorded (if known)
DE 37 Actual age in years. 00 = Unknown. NB = Newborn (under 24 hrs). BB = Baby (under 1 year).
Offender Sex recorded (if known)
DE 38 M, F, or U Unknown.
Offender Race recorded (if known)
DE 39 Same codes as victim race. Use U if unknown.
Offender Ethnicity recorded (if known)
DE 39A H, N, or U.
Arrest Made Arrestee Segment ▶
Complete this section only when an arrest was made. Group B offenses (DWI, disorderly conduct, etc.) only generate a LIBRS record when an arrest occurs.
Arrest Date is on or after the incident date
DE 42 Cannot precede the incident date. Verify date is accurate.
Type of Arrest coded
DE 43 O On-View, S Summoned/Cited, T Taken Into Custody.
Multiple Arrestee Indicator completed
DE 44 M Multiple (one person arrested for multiple offenses in this incident), C Count Arrestee (individual counted only once per incident), N Not Applicable.
UCR Arrest Offense Code matches incident offense
DE 45 The arrest offense code must be a valid NIBRS code and should match the offense(s) in the incident.
Arrestee Armed With coded
DE 46 Select all weapons the arrestee had at time of arrest. Use 01 Unarmed if no weapon. Do not leave blank.
Arrestee Age, Sex, Race, Ethnicity, Resident Status recorded
DE 47–51 All five demographics required. Use U for any unknown field.
Disposition of Juvenile Arrestee (if under 18)
DE 52 Required when arrestee age < 18. Options: D Handled & Released, J Juvenile Court, R Referred to Other Authority, S Supervised Release, U Criminal Court.

Group B Offenses — Arrest Required
Group B offenses (DWI, Disorderly Conduct, Liquor Law Violations, Curfew Violations, Runaway, Trespass, Vagrancy, Peeping Tom, Family Offenses Non-Violent) are only submitted to LIBRS when an arrest is made. If no arrest, no LIBRS record is generated for Group B offenses. The Arrestee Segment is required for all Group B submissions.
Supervisor Supervisor Review Checklist ▶
Use this section when reviewing a subordinate's completed report prior to LIBRS submission. Cross-reference with the NIBRS Code Verification Tool for all charged statutes.
NIBRS Code Verification
Each LRS statute has been verified against the NIBRS Verification Tool
Confirm every charged statute maps to the NIBRS code selected in the report. Pay special attention to sexual offenses, homicide statutes, and robbery — these have the highest misclassification rate.
NIBRS code is current (not expired)
Verify the LRS-to-NIBRS mapping is still active as of the incident date. Expired mappings will be flagged by LCLE.
Multi-NIBRS statutes: most appropriate code selected
When a statute maps to more than one NIBRS code, confirm the officer chose the code that best reflects the actual conduct, not simply the first option.
Data Completeness
Aggravated Assault Circumstance (DE 31) present for all 13A/09A/09B/09C charges
The single most common rejection reason in BRPD submissions. Verify it is not blank.
Method of Entry present for all burglary (220) charges
Second most common rejection. Must be Force or No Force — never blank.
All stolen/damaged property has a value greater than $0
$0 values on stolen property cause submission errors. Verify each property entry has a realistic estimated value.
Victim-offender relationships coded for all crimes against persons
Critical for domestic violence, sexual offense, and assault cases. Affects FBI statistics on intimate partner violence.
Incident date falls within the current reporting period
Incidents from prior months appearing in the current submission create reporting period mismatch warnings at LCLE.
Drug type, quantity, and measurement entered for all drug charges
35A and 35B offenses require all three drug data fields. Blank drug data is a frequent gap in drug-related submissions.
Group B offenses only submitted when arrest was made
Confirm no Group B-only incidents are submitted without an accompanying Arrestee Segment.
Final Approval
Report narrative is consistent with NIBRS codes and data elements
The narrative should support the offense classification. Discrepancies between the narrative and coded data can trigger audits.
Report approved for LIBRS submission
Supervisor signature / badge number and date of review.
